Java_JDBC

﻿JDBC

是Java编写的程序不依赖与具体的数据库，操作数据库的API

应用程序 —–> JDBC—–> 数据库

  • 与一个数据库建立连接
  • 向已连接的数据库发送SQL语句
  • 处理SQL语句返回的结果

MySQL数据库 SQL语句（详情请点击）

连接数据库

加载JDBC-数据库驱动（连接器） JDBC调用本地的JDBC-数据库驱动和相应的数据库建立连接

Java应用使用JDBC API访问数据库基本工作

  • 加载JDBC-数据库驱动 //Class.forName("com.mysql.jdbc.Driver");
  • 和指定的数据库建立连接 //Connection con=DriverManager.getConnection(url,user,password);
  • 创建操作数据库SQL的Statement、PreparedStatement对象
  • 执行语句分析执行结果
  • 关闭连接

1. 下载JDBC-MySQL数据库驱动

下载、加载 mysql-connector-java-5.1.40-bin.jar ，即 Java-JDBC-MySQL数据库驱动
放置在JAVA_HOME/jre/lib/ext/下，再复制一份放到jdk同路径的jre/lib/ext下
此目录下为jre扩展，放置不是Java运行环境类库中的 外部添加类

封装在Driver类中，使用压缩工具打开可以看到 Driver.class 文件
想要看Driver.class文件源码，可以使用jad.exe文件反编译出源码

2. 测试数据库操作

  • 以 message 为 测试数据库

java.sql包中的DriverManager类有两个用于建立连接的类方法

    • static Connection getConnection(java.lang.String,java.lang.String,java.lang.String)
    • static Connection getConnection(java.lang.String)

    返回一个Connection对象

使用1类方法

使用2类方法

数据库中有汉字，需要额外参数 characterEncoding
String uri="jdbc:mysql://192.168.100.1:3306(DB服务器驻留计算机IP地址)/<databasename>?useSSL=true&characterEncoding=utf-8"
characterEncoding参数值为utf-8，数据库建立时也记得设置字符集为utf8

连接成功后，可以通过JDBC提供的API和数据库，用SQL语句获取 查询、更新操作数据
所有完成结束后，常记con.close() //关闭数据库

  • Statement声明一个SQL语句对象，con调用方法createStatement()返回该对象
  • SQL查询结果返回给一个ResultSet对象

ResultSet对象和连接对象Connection 绑定，连接对象关闭，ResultSet对象数据立刻消失，也就是在关闭前输出数据。

ResultSet对象使用方法
getString(int columnindex)
getString(String columnName)
boolean next() //游标移动到下一数据行

rs对象，游标位于第一行前面，next()方法向下移动游标

若需要在结果集ResutlSet对象中上下移动

Statement sql=con.createStatement(int type,int concurrency);

根据参数type、concurrency取值，返回一个可滚动的、相应类型的结果集  //不介绍了

  • Statement sql 对象更新、添加、删除操作

    ​ public int executeUpdate(String sqlStatement);

返回的 int 值与更新数据记录个数有关，不返回变量也可以，直接执行sql.executeQuery(sqlStr)

最常用

- 预处理语句对象
意思就是sql预处理
不是连接后再由 数据库本身sql语句解释器 解释为数据库底层内部命令
改变为 JDBC 预处理
prepareStatement presql=con.prepareStatement(sqlStr);
