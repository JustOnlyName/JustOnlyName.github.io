JSP

JSP 与 PHP、ASP、ASP.NET 等语言类似，运行在服务端的语言。
全称Java Server Pages，可以响应客户端请求，嵌入文档中而动态生成 HTML或其他格式文档的Web网页

  jsp属于动态网页，动态网页随处可见。比如搜索引擎，会将相关搜索结果呈现在页面上。
  此时服务器端其实进行了一次搜索工作，人工无法完成，搜索过程由程序完成，进行查询数据库。
  单纯的静态页面 html 无法完成动态查询数据库操作，但Java代码可以
  在 html 代码中混合Java代码，就能让网页具有动态功能。
  由 JSP 容器执行该页面中的Java代码，实时生成的 html 页面由服务端向客户端发送。

问题

  • Tomcat启动一闪而过就消失的原因和解决方法

    一些一闪而过的bat文件，可以使用cmd窗口打开运行，这样可以有回显，查看问题所在

    在cmd窗口进入到tomcat\bin目录，启动startup.bat，显示
    Neither the JAVA_HOME nor the JRE_HOME environment variable is defined
    At least one of these environment variable is needed to run this program

    主要是同一目录下的 setclasspath.bat文件造成的， 编辑程序打开看一看就能懂
    设置两个环境变量即可

    其中JRE_HOME指的是 jdk 路径下的jre目录，不是外层的那个

    还有一个问题，就是常用到关于tomcat 的计算机服务管理，但在列表中没有Apache Tomcat
    进入安装bin目录，cmd运行，service.bat install

    之后并没有配置JRE_HOME 仍热能正常运行

JSP 指令 page

编译过程

  • 第一阶段是jsp编译成.java：根据pageEncoding的设定读取jsp，由指定的编码方案翻译成统一的UTF-8 java 源码
  • 第二阶段是由JAVAC用UTF-8的encoding读取java源码，编译成UTF-8 encoding的二进制码(.class)，不论JSP编写时候用的是什么编码方案，经过这个阶段的结果全部是UTF-8 编码
  • 第三阶段是Tomcat 载入和执行 class 码，输出的结果呈现在客户端。这时隐藏在阶段一和阶段二的参数contentType就发挥了功效

简单来讲，要经过 两次 编码过程

第二阶段中，.jsp 第一次被请求时，编译 .class 文件缓存在特定的目录中。下一个同样的请求将直接把缓存的结果返回给用户。
