14-kalinux

Posted on 2021-01-16 Edited on 2021-01-17

因为下午准备写一些博客，但是打开Windows 之后出好多问题

  1. Windows Denfender 的自动开启变成了手动开启，而且计算机服务WD 的属性中下拉菜单也是灰色无法更改
  2. 所有程序打开都变慢了很多，有的根本打不开，关机的开始菜单都打不开。好不容易关了机，打算进kali 看看能做点什么

我这里kali 是2020.1b 版本的，官方解决中文乱码的那个版本。使用的Xfce4 桌面。

最初选择入2020 以后的版本，是看上了Xfce4 新出的能够使用到虚拟的Win 界面。外表上就像是使用Win 系统，实际上是在使用kali。测试之后感觉……没什么用，还不如直接考虑美化kali 界面呢

设置Win 常用快捷键

常使用的Windows 中的快捷键，比如剪贴板的 Win + V、文件管理器的 Win + E

但是在kali 中并没有这些个快捷键，需要自己添加。

可以仿照类似 terminal 的Ctrl + Alt + T 的快捷键，在
Settings - 键盘 - 快捷键 中添加

文件管理器 /usr/bin/thunar

剪贴板 的程序比较多，但是类似Windows 中的，查看并选择历史剪贴内容的，不是clipboard 程序本身，而是那个历史清单 /usr/bin/xfce4-clipman-history

显示桌面 默认是Ctrl + Alt + D 想要更改为Win + D ，发现不是在键盘 - 快捷键 中设置。
  实际上，显示桌面 这个操作没有归类到与desktop 相关，而是认为将所有窗口最小化。在Settings - 窗口管理器(Window Manager) 里面设置更改即可。

蜂鸣声

关闭kali linux xfce4 的蜂鸣声

之前一直以为是点关机键、输入框里面没有内容，按左方向向左移动光标时候会响

如今在登录界面向左移动光标到头也会响，而且还是在执行了 xset b off 之后。感觉就像是登陆完成后，这条指令才生效，登陆界面不管用

现在有一个方法，两种情况都解决
cd /etc/modprobe.d/
ls

看到有两个文件，在这两个文件中最后添加一句
blacklist pcspkr

关于root 权限

我一直以为，安装kali 之后，用的自己用户的密码，和root 用户的密码应该是一样的
现在才知道，root 的密码是不知道的

那我之前一直sudo sudo 的，这是什么回事

临时

其实这样输入，只需要的是当前管理员用户密码（其实如果就自己用户一个，可以认为自己用户就是管理员用户）
就可以得到超级用户的权限。但默认的情况下5分钟root权限就失效了。

这样已经让我感觉到一丝麻烦，有时候过一会儿需要sudo 一下，又要输入密码，很耽误事，万一有时候还会输入错误

那么，如何使用sudo 命令不用输入密码

一、

使用root 用户修改

二、
在/etc/sudoers.d/ 目录下创建一个文件，如add_sudo_nopasswd
写入
%sudo ALL=(ALL:ALL) NOPASSWD:ALL

一、二 两种方式效果一样，使用一种即可。我选择了第二种

非临时

还有一种进入root 用户的角色，而不是临时使用root 角色执行命令

  1. sudo -i 输入密码即可
  2. su 切换用户，但需要先临时使用root 用户更改我们不知道的root 密码
    sudo passwd root 可以设置了
    这样是为了可以登陆界面里 除了自己用户，还可以输入root 的

终端内容颜色

换用root 用户之后，终端中root 区域的字符颜色全是白色的，更改为像是自己用户一样的颜色

压缩包

安装本地下载的.deb 软件包，一般使用比较繁琐的dpkg

双系统 windows 无法启动（no argument specified）

vim /boot/grub/grub.cfg
grub.cfg中windows那一段中的–set改成–set=root就OK了

修改启动分辨率：

感觉启动引导菜单和tty的分辨率比较蛋疼？按照如下方法修改即可：

（root权限运行）

  1. 使用编辑器编辑/etc/default/grub 这个文件

比如我使用vim

vim /etc/default/grub

找到 #GRUB_GFXMODE=640×480 取消这一行前面的注释符“#”并将后面的数字修改为一个合适的值，不需要太高，比如1024×768。这个值同时会影响grub启动菜单和控制台里文字的分辨率。

  1. 修改 /etc/grub.d/00_header 这个文件

vim /etc/grub.d/00_header
查找关键字”set gfxmode=${GRUB_GFXMODE}”(去掉双引号)，然后在这行下面添加新行， 内容是：”set gfxpayload=keep” (去掉双引号)

  1. 更新Grub配置

update-grub
